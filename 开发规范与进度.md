## 一、项目定位

多智能体医疗会诊系统(论文第四章)。技术栈:**Streamlit + FastAPI + SQLite + FAISS**,推理内核为第三章微调后的 **MoE-LoRA + 困惑度驱动动态秩** 模型。阶段 0~6 使用 mock 模型跑通系统层,阶段 7 接入真实模型。

## 二、硬性开发规范(违反会破坏论文亮点,**必读**)

### R1. 一 4.3 节 一 目录/文件
`app/coordinator / app/router / app/agents / app/modes / app/aggregator / app/safety / app/knowledge / app/storage / app/model` 与论文 4.3.1~4.3.6 严格一一对应,**禁止合并**。答辩时需要"指着代码说这就是某一小节"。

### R2. 协调器是唯一大脑
- 只有 `ConsultationOrchestrator` 可以编排其他模块;router / agents / aggregator / safety **不得互相直接调用**。
- **数据库只由协调器读写**。其他模块不得直连 `app/storage/`。
- 违反会导致状态一致性崩盘(论文 4.3.1 原话)。

### R3. 模型层与系统层严格隔离
- `app/model/` 对外仅暴露 `ModelEngine.generate(prompt, rank_hint=None) -> (text, meta)`。
- `app/agents/` 只负责构造 prompt、解析输出,**绝不触碰模型内部**。
- 换基座模型时只需改 `app/model/`,上层零改动。

### R4. 两层专家是"软对应",不是强绑定
- 系统层的科室智能体**只能**通过 prompt 注入"科室名 + 推理重点"。
- **严禁**出现"请激活第 X 号专家"、"请使用科室 Y 的路由"这类指令。
- 模型层 `MoELoRALinear.layer_router` 的激活权重完全由隐状态决定,不接受外部科室标签控制。
- **这是第四章核心亮点,写错整章设计坍塌**。

### R5. 仲裁动作发起方必须是协调器
- `aggregator` 命中 L3 时通过 `coordinator_hook` 回调协调器。
- 协调器调用 `model.generate(prompt, rank_hint=max(rank_bins))` 获得仲裁稿。
- 代码里**必须**出现显式 `rank_hint=max(...)` 参数,否则"系统层与模型层协同"亮点落不了地。

### R6. 路由置信度公式
- 按 `dept` 分组 → 取该科室下相似度 **最大值**(**不是均值**!)→ softmax。
- 一致性检验用 **余弦最小值**(非均值),医疗场景对冲突零容忍。
- L2 加权中 `relevance` / `confidence` **直接读 `RoutingResult`,不要重算**;`completeness` 是五字段覆盖度线性扣分。

### R7. 会话 = 病例
一次 session 绑定一个病例,**禁止中途换病人**(极大简化状态管理)。

### R8. 异步边界
- FastAPI 是 async,但 `model.generate()` 是同步,必须 `await asyncio.to_thread(model.generate, ...)` 包一层。
- **不得用线程池**,会阻塞 FastAPI 事件循环。
- 并行模式用 `asyncio.gather + asyncio.wait_for`。

### R9. 配置外置
- 阈值 / 提示词 / 安全规则全部放 `config/*.yaml`,**不得硬编码到 `.py`**。
- 答辩时"为什么这么设计提示词"可以现场打开 YAML。

### R10. `inference_meta` 要存全
`{rank, router_weights, elapsed_ms, …}` 每次推理都写入 message 表。这是 4.5.6 案例分析与专家分化可视化的**唯一数据源**,漏存无法补。

### R11. 安全降级不是拒绝
替换风险表述 + 保留非风险结构 + 追加强化免责声明,三步缺一不可。免责声明用固定模板字符串,**不走 LLM**。

## 三、代码风格约定

| 项 | 约定 |
| --- | --- |
| 语言 | Python 3.10+,类型注解覆盖公共 API |
| 数据契约 | Pydantic v2,定义在 `app/schemas/` |
| 异步 | async def + `asyncio`;禁止 `threading` 做并发推理 |
| 日志 | `app.utils.logger.get_logger(name)` 输出 JSON 行 |
| 错误处理 | 系统边界校验输入;内部函数不加无意义的 try/except |
| 未实现 | `raise NotImplementedError("Phase N")` 并在 docstring 中写明阶段 |
| 注释 | 用简体中文;论文公式对应处用 `# 论文 4.x.y` 标注 |
| 测试 | `pytest`;异步测试加 `pytest-asyncio` |
| 配置加载 | 通过 `yaml.safe_load` 读 `config/*.yaml`,**不要用 `yaml.load`** |

## 四、环境与工具链

- **Python 环境**:conda env `qwen`(用户自备,已含部分依赖)。本仓库**只维护 `requirements.txt`**,**不执行 `pip install`**;需要补依赖时由用户手动安装。
- **依赖安装(用户按需执行)**
  ```bash
  conda activate qwen
  pip install -r requirements.txt
  ```
- **嵌入模型**:默认 `BAAI/bge-base-zh-v1.5`。首次使用需联网下载 ~400MB。若无外网,把 `config/settings.yaml` 的 `embedder.model_name_or_path` 改成本地目录路径。
- **FAISS 索引**:`data/faiss_index/{faiss.index, docs.jsonl}`,被 `.gitignore` 忽略,运行 `scripts/build_kb_index.py` 生成。
- **SQLite**:`data/consultation.db`,被 `.gitignore` 忽略,运行 `scripts/init_db.py` 生成。

| `FinalReport` | 最终报告 | `summary / dept_opinions / aggregation_level(1-3) / safety_action(pass/degraded/arbitrated) / disclaimer` |
